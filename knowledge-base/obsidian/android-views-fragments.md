# Android Views And Fragments

## Что используется

Импортированный Android-клиент построен на классическом Android UI:

- `Activity`
- `Fragment`
- XML layouts
- `BottomNavigationView`
- `RecyclerView`
- View Binding

## Почему это важно

Это не Compose-приложение. Значит, документацию и отчёт нужно писать под классический Android UI стек.

## Текущая структура экранов

- `MainActivity` - вход
- `SecondActivity` - контейнер нижней навигации
- `ProfileFragment` - профиль пользователя
- `StatisticsFragment` / `MyStatFragment` - статистика
- `StockListFragment` - список акций
- `StockChartFragment` - карточка акции и графики

## Сильные стороны

- UI уже разбит на отдельные экраны
- навигация понятная
- данные по котировкам загружаются асинхронно

## Ограничения

- вход пока локальный, без реального backend
- buy/sell ещё не подключены
- часть статистики и графиков работает на моках

## Что нужно помнить

Для интеграции с сервером придётся добавить сетевой слой, DTO и хранение access token, но сама структура экранов уже подходит для этого.
