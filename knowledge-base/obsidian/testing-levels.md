# Testing Levels

## Что требует курс

В отчёте нужно описать три уровня тестирования:

- модульное
- интеграционное
- системное

## Как это переносится на проект

### Модульное тестирование

Проверяются отдельные классы и use case:

- регистрация
- расчёт статистики
- логика продажи
- парсинг котировок

### Интеграционное тестирование

Проверяются связки подсистем:

- HTTP routes + use case
- репозитории + тестовая БД
- gateway + upstream
- data layer Android-клиента + mock server

### Системное тестирование

Проверяется полный пользовательский сценарий:

- логин
- получение котировки
- покупка акции
- просмотр статистики
- появление следов в observability stack

## Текущее состояние

- у `mobile-backend` есть хороший старт автоматизированных тестов
- у `quotes-service` есть unit/integration тесты
- `mobile-api` и `android-app` в тестировании заметно отстают

## Итог

Тестовая стратегия уже понятна, но зрелость модулей пока разная. Это нормально для проекта, который развивался по частям и собирался из нескольких источников.
