# Redis Streams And ClickHouse

## Почему в проекте два разных хранилища

Они решают разные задачи.

## Redis

Используется для:

- `Streams` с доменными событиями
- latest-cache по котировкам
- лёгкой интеграции между сервисами

### Почему Redis удобен

- быстрый
- подходит для событий и кеша
- прост в локальном запуске через Docker

## ClickHouse

Используется для:

- хранения истории котировок
- быстрых аналитических запросов по временным рядам

### Почему ClickHouse удобен

- хорошо подходит под большие объёмы append-only данных
- оптимизирован под аналитические чтения

## Как это ложится на проект

- `quotes-service` пишет latest-state в Redis и историю в ClickHouse
- `mobile-backend` публикует события пользователей и сделок в Redis Streams

## Что важно понимать

PostgreSQL хранит транзакционную часть бизнеса, а Redis и ClickHouse закрывают интеграционно-аналитические задачи. Это нормальное разделение обязанностей, а не дублирование данных.
