# OpenTelemetry Basics

## Что это

OpenTelemetry - стандартный способ собирать телеметрию: traces, metrics и logs.

## Термины

- `trace` - полный путь одного запроса через систему
- `span` - отдельный шаг внутри trace
- `collector` - промежуточный сервис, который принимает и перенаправляет телеметрию
- `OTLP` - протокол передачи телеметрии

## Как это устроено у нас

- `mobile-api` создаёт server spans на HTTP-запросы
- `mobile-backend` может записывать внутренние spans через `TelemetryRecorder`
- `quotes-service` экспортирует телеметрию через Go SDK
- `otel-collector` принимает данные и отправляет traces в `tempo`

## Зачем это нужно

- видеть путь пользовательского запроса между сервисами
- искать узкие места в latency
- быстрее разбирать ошибки в распределённой системе
